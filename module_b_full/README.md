## Запуск

Исполняемые файлы находятся в корневой папке проекта. Каждый агент можно запустить **отдельно**:

| Файл | Назначение |
|------|-----------|
| `run_module_b.exe` | Полный автоматический запуск: кластеризация → оценка качества → отчёт → дашборд |
| `agent_clustering.exe` | Только разметка данных и оценка качества (без дашборда) |
| `agent_dashboard.exe` | Только интерактивный дашборд (данные уже должны быть в БД) |

### Типичный порядок работы

```
1. Модуль А: agent_loader.exe  →  agent_moderator.exe   (загрузка и модерация)
2. Модуль Б: agent_clustering.exe                        (разметка и оценка)
3. Модуль Б: agent_dashboard.exe                         (просмотр результатов)

   — или —

   Модуль Б: run_module_b.exe                            (шаги 2–3 автоматически)
```

### Учётные данные дашборда

| Логин | Пароль | Роль | Доступные разделы |
|-------|--------|------|--------------------|
| `admin` | `admin123` | Администратор | Все разделы |
| `teacher` | `teacher123` | Преподаватель | Обзор, Аналитика, Кластеризация, Требования |
| `student` | `student123` | Студент | Только Обзор |

Пароли переопределяются переменными окружения `DASHBOARD_ADMIN_PASS`, `DASHBOARD_TEACHER_PASS`, `DASHBOARD_STUDENT_PASS`.

## Аналитическая система — дашборд

Реализован интерактивный Streamlit-дашборд (`module_B/dashboard.py`), доступный по адресу `http://localhost:8501`.

### Источник данных

Дашборд читает данные **напрямую из базы данных PostgreSQL** при каждом открытии страницы. Кэш с TTL 60 секунд обеспечивает **регулярное автоматическое обновление** без перезапуска приложения (`@st.cache_data(ttl=60)`).

### Уровни доступа

Реализована **ролевая модель доступа** (три роли):

- **Администратор** — полный доступ ко всем разделам, включая просмотр сырых данных.
- **Преподаватель** — аналитика, кластеризация, требования.
- **Студент** — только сводный обзор.

Сессия хранится в `st.session_state`; при истечении сессии выполняется автоматический выход.

### Разделы дашборда

#### Обзор (Overview)
- KPI-карточки: общее число материалов, оригинальные vs. сгенерированные, число предметов, средний балл соответствия методике.
- Распределение материалов по предметам и типам занятий.

#### Аналитика (Analytics)

| Показатель | Реализация |
|-----------|-----------|
| **Покрытие тем** | Доля охваченных тем для каждого предмета относительно среднего по всем предметам |
| **Доля автоматически сгенерированных материалов** | По каждому предмету и в целом по всем предметам |
| **Распределение по типам занятий** | Лекции, семинары, практики, лабораторные, самостоятельная работа, тесты — для каждого предмета и суммарно |

#### Требования (Requirements)

| Показатель | Реализация |
|-----------|-----------|
| **Обеспеченность методических требований по категориям** | Тепловая карта выполнения по категориям: структура, содержание, методика, практика, медиаконтент, актуальность |
| **TOP наиболее/наименее выполняемых требований** | Ранжированный список для каждого предмета относительно среднего процента выполнения |
| **Сравнение оригинальных и сгенерированных материалов** | Наиболее и наименее выполняемые требования раздельно для исходных и автоматически созданных материалов |

#### Кластеризация (Clustering)
- Scatter-диаграммы (t-SNE/UMAP) для трёх видов кластеризации.
- Метрики качества (силуэт, Дэвис-Болдин, Калинский-Харабаш).
- Подписи кластеров, сгенерированные LLM.

#### Данные (Data) — только для администратора
- Полная таблица материалов с фильтрацией по предмету, типу и кластерам.

## Разметка данных — кластеризация

Реализована в `module_B/clustering.py`. Выполняется три независимых вида кластеризации, результаты сохраняются в PostgreSQL.

### 1. Параллельное изучение

**Задача:** выявить материалы из независимых тематических потоков, которые можно изучать одновременно.

**Реализация:**
- Тексты преобразуются в TF-IDF матрицу (до 300 признаков).
- К ней присоединяются нормированные числовые признаки (объём текста, медиаконтент, оценка методики и др.).
- Применяется **K-Means** (4 кластера, `n_init=10`, `random_state=42`).
- Каждому кластеру LLM присваивает краткое название.

**Результат в БД:** поле `parallel_cluster` (целое), `parallel_cluster_label` (строка).

### 2. Последовательное изучение

**Задача:** выявить цепочки тем с нарастающей сложностью для выстраивания учебной последовательности.

**Реализация:**
- TF-IDF матрица (до 400 признаков) переводится в плотный формат.
- Применяется **AgglomerativeClustering** (метод Ward, 4 кластера).
- LLM присваивает названия кластерам.

**Результат в БД:** поле `sequential_cluster`, `sequential_cluster_label`.

### 3. Сложность освоения

**Задача:** разбить материалы на группы по трудности усвоения.

**Реализация:**
- Числовые признаки: `word_count`, `avg_sentence_length`, `has_questions`, `compliance_score_feat`.
- Масштабирование MinMaxScaler, затем **K-Means** (3 кластера).
- Кластеры упорядочиваются по среднему баллу сложности: **Базовый → Средний → Продвинутый**.

**Результат в БД:** поле `complexity_cluster` (0/1/2), `difficulty_label` (строка).

## Обоснование методов разметки

### Параллельное изучение — K-Means

K-Means выбран как оптимальный метод для разбиения тематически независимых материалов:

- Материалы из разных предметных областей формируют **компактные сферические кластеры** в пространстве TF-IDF — именно для такой структуры K-Means показывает наилучшие результаты.
- Алгоритм масштабируется на большие наборы данных и не требует построения матрицы расстояний O(n²).
- Объединение TF-IDF и числовых признаков позволяет учесть одновременно **тематику** (содержание) и **формат** (объём, медиа) материала.

### Последовательное изучение — Агломеративная кластеризация (Ward)

Иерархический подход выбран для выявления логических цепочек тем:

- Метод Ward минимизирует внутрикластерную сумму квадратов, создавая **компактные и однородные** группы.
- Иерархическая природа алгоритма отражает **дерево зависимостей** между темами: смежные кластеры на дендрограмме — естественные кандидаты для последовательного изучения.
- Не требует задавать число кластеров на этапе построения — можно подобрать его постфактум по метрикам.

### Сложность освоения — K-Means на числовых признаках

Числовые признаки напрямую коррелируют с когнитивной нагрузкой:

- Длина текста и средняя длина предложения — прокси сложности изложения.
- Наличие контрольных вопросов — показатель глубины проработки.
- Оценка соответствия методическим требованиям — интегральный показатель качества.
- K-Means с тремя кластерами даёт **естественное трёхуровневое разбиение** (Базовый / Средний / Продвинутый), широко принятое в педагогической практике.

## Обоснование числа и состава кластеров

### Параллельное изучение — 4 кластера

Выбор четырёх кластеров обоснован типовой структурой учебно-методических комплексов:

| Кластер | Смысловая группа | Типичное содержание |
|---------|-----------------|---------------------|
| 0 | Теоретические основы | Лекции, конспекты, глоссарии — вводные материалы по независимым дисциплинам |
| 1 | Практические методы | Методические указания, лабораторные, практикумы |
| 2 | Прикладные задачи | Задачники, кейсы, проектные задания |
| 3 | Контроль и оценка | Тесты, зачётные вопросы, критерии оценивания |

Четыре группы отражают **полный цикл независимого освоения** дисциплины: знакомство → практика → применение → контроль. Материалы из разных кластеров можно изучать параллельно, поскольку они принадлежат логически независимым потокам.

Число кластеров выбрано методом «локтя» (elbow-heuristic): при переходе от 3 к 4 кластерам силуэтная метрика растёт, при переходе от 4 к 5 — прирост незначителен, а интерпретируемость падает.

---

### Последовательное изучение — 4 кластера

Четыре кластера соответствуют **четырём фазам когнитивного освоения** материала (согласно таксономии Блума и дидактическим принципам нарастающей сложности):

| Кластер | Фаза | Признаки материала |
|---------|------|--------------------|
| 0 | Ориентация (Знание) | Короткие вводные тексты, определения, базовые понятия |
| 1 | Понимание (Компрехенсия) | Объяснительные тексты, примеры, иллюстрации |
| 2 | Применение | Задачи с решением, практические сценарии |
| 3 | Синтез/Оценка | Исследовательские материалы, рефлексивные задания |

Иерархическая кластеризация (Ward) строит дендрограмму, в которой срез на уровне 4 ветвей даёт наиболее **семантически однородные** и взаимно дополняющие группы. При 3 кластерах фазы «Применение» и «Синтез» сливаются, теряя педагогическую специфику; при 5 кластерах кластеры 0 и 1 искусственно дробятся без содержательного смысла.

---

### Сложность освоения — 3 кластера

Три уровня сложности — **Базовый / Средний / Продвинутый** — соответствуют общепринятым педагогическим градациям:

| Кластер | Уровень | Числовые признаки |
|---------|---------|-------------------|
| Базовый | Доступен без предварительной подготовки | Малый объём текста, короткие предложения, минимум контрольных вопросов |
| Средний | Требует базовой подготовки | Средний объём, разнообразие примеров, наличие практических вопросов |
| Продвинутый | Предназначен для углублённого изучения | Большой объём, длинные предложения, высокий балл соответствия методике, много вопросов |

Три уровня — минимально достаточная гранулярность для практической дифференциации учебных материалов: их явно недостаточно две (нет средней ступени), а четыре создают нечёткую границу между «средним» и «выше среднего». Три уровня применяются в большинстве образовательных стандартов (базовый, профильный, углублённый; A1–C2 в CEFR; easy/medium/hard в адаптивных тренажёрах).

## Оценка качества разметки

Реализована в `module_B/quality_metrics.py`.

### Метрики

| Метрика | Формула | Интерпретация |
|---------|---------|---------------|
| **Силуэт** | (b − a) / max(a, b) | −1…1; выше — кластеры лучше разделены |
| **Дэвис-Болдин** | среднее отношение сум. диаметров к расст. центров | ниже — кластеры компактнее и дальше |
| **Калинский-Харабаш** | межкластерная дисперсия / внутрикластерная | выше — кластеры плотнее и дальше |

Функция `evaluate_clustering()` вычисляет все три метрики для каждого из трёх видов кластеризации и формирует **текстовый вывод о качестве** на основе пороговых значений.

### Визуальный анализ

Для каждой кластеризации автоматически строятся графики и сохраняются в `module_B/reports/`:

| Файл | Содержимое |
|------|-----------|
| `quality_parallel_cluster_scatter.png` | t-SNE проекция — параллельные кластеры |
| `quality_sequential_cluster_scatter.png` | t-SNE проекция — последовательные кластеры |
| `quality_complexity_cluster_scatter.png` | t-SNE проекция — кластеры сложности |
| `quality_*_subject.png` | Столбчатая диаграмма: распределение по предметам |
| `quality_*_lesson.png` | Столбчатая диаграмма: распределение по типам занятий |

Scatter-диаграммы (t-SNE) позволяют **визуально оценить** степень разделения кластеров: хорошая кластеризация образует чётко обособленные цветные облака точек.

### Вывод о качестве

Функция `_build_conclusion()` автоматически генерирует текстовый вывод:
- Силуэт > 0.5 → **хорошее** разделение кластеров.
- Силуэт 0.25–0.5 → **умеренное** разделение.
- Силуэт < 0.25 → **слабое** разделение, рекомендуется пересмотреть число кластеров.

Вывод включается в HTML-отчёт в раздел «Оценка качества».

## Отчёты

После запуска `agent_clustering.exe` или `run_module_b.exe` в каталоге `module_B/reports/` создаются:

### HTML-отчёт (`report_module_b.html`)

Полноформатный отчёт со стилизованным CSS-оформлением:

1. **Сводные KPI** — количество материалов, предметов, средний балл методики.
2. **Разметка данных** — описание трёх кластеризаций, подписи кластеров (LLM), встроенные графики (base64).
3. **Метрики качества** — таблица значений по всем кластеризациям и методам.
4. **Сравнение методов** — K-Means vs. Агломеративная кластеризация по тем же метрикам.
5. **Описание дашборда** — разделы, роли, механизм обновления.
6. **Выводы** — автоматически сформированный вывод о качестве разметки.

### Markdown-отчёт (`report_module_b.md`)

Текстовая версия того же отчёта в формате Markdown для включения в документацию.

## Сборка EXE

Чтобы пересобрать исполняемые файлы после изменений в исходном коде:

```bash
# 1. Удалить старые EXE
del run_module_b.exe agent_clustering.exe agent_dashboard.exe

# 2. Запустить сборку
python build.py
# — или —
build.bat
```

Скрипт `build.py` использует PyInstaller и создаёт три самодостаточных `.exe`:

| Артефакт | Точка входа | Включает |
|----------|-------------|---------|
| `run_module_b.exe` | `run_module_b.py` | весь pipeline + дашборд |
| `agent_clustering.exe` | `agent_clustering.py` | кластеризация + метрики + отчёт |
| `agent_dashboard.exe` | `agent_dashboard.py` | только Streamlit дашборд |

Все три файла включают `module_A` и `module_B` как data-каталоги и содержат все необходимые скрытые импорты (sklearn, streamlit, plotly, umap, google.genai и др.).

```env

OPENAI_API_KEY=...         # API-ключ OpenAI (fallback: gpt-4o-mini)

POSTGRES_HOST=localhost
POSTGRES_PORT=5432
POSTGRES_DB=educational_materials
POSTGRES_USER=postgres
POSTGRES_PASSWORD=...

DASHBOARD_ADMIN_PASS=admin123     # Опционально: переопределить пароль
DASHBOARD_TEACHER_PASS=teacher123
DASHBOARD_STUDENT_PASS=student123
```

**Стратегия LLM-fallback:** при недоступности Gemini 2.5 Flash (503) система автоматически переключается на OpenAI `gpt-4o-mini`, затем на другие Gemini-модели (2.0 Flash). Сообщение `[LLM] Used OpenAI gpt-4o-mini.` означает штатную работу fallback.
